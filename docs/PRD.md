# PRD — แอปพลิเคชันตรวจแปลงเกษตร
### Mega Farm Cambodia — Field Inspection App

**เวอร์ชัน:** 1.0  
**วันที่:** 2026-06-18  
**เจ้าของผลิตภัณฑ์:** Mega Farm Cambodia

---

## 1. ภาพรวมผลิตภัณฑ์

ระบบช่วยเจ้าหน้าที่ภาคสนามตรวจสอบสภาพพืชในแปลง โดย:

1. รับภาพพืช + พิกัด GPS + เวลา
2. ดึงข้อมูลสภาพอากาศ ณ พิกัดนั้นอัตโนมัติ
3. ส่งให้ AI วินิจฉัยสาเหตุ 5 หมวด พร้อมแหล่งอ้างอิง
4. บันทึกผลทั้งหมดลง Google Drive ของฟาร์ม

---

## 2. ผู้ใช้งาน

| กลุ่ม | บทบาท |
|-------|-------|
| เจ้าหน้าที่ภาคสนาม | ถ่ายรูปพืช บันทึกพิกัด ส่งวิเคราะห์ |
| ผู้จัดการฟาร์ม | ดูรายงานสรุป ติดตามแนวโน้มปัญหา |
| นักวิชาการเกษตร | วิเคราะห์ข้อมูลย้อนหลัง |

---

## 3. ฟีเจอร์หลัก

### F-01 รับภาพพืช
- รองรับไฟล์ `.jpg` `.png` `.webp`
- รองรับสูงสุด 5 ภาพต่อการตรวจ 1 ครั้ง
- บีบอัดภาพอัตโนมัติก่อนส่ง AI (ไม่เกิน 1920px)

### F-02 พิกัด GPS
- รับค่า `latitude`, `longitude` จากอุปกรณ์
- บันทึก `gps_accuracy_m` เพื่อระบุความน่าเชื่อถือ
- ปฏิเสธพิกัดที่ accuracy แย่กว่า 500 เมตร

### F-03 ดึงเวลาอัตโนมัติ
- บันทึก timestamp (ISO 8601, timezone Asia/Phnom_Penh)
- ไม่อนุญาตแก้ไขเวลาด้วยมือ

### F-04 ดึงสภาพอากาศจากพิกัด (Open-Meteo)
ดึงข้อมูลอัตโนมัติโดยไม่ต้องให้เจ้าหน้าที่กรอก:

| ข้อมูล | หน่วย |
|--------|-------|
| อุณหภูมิ | °C |
| ความชื้นสัมพัทธ์ | % |
| ปริมาณฝน (1 ชั่วโมงล่าสุด) | mm |
| ความเร็วลม | km/h |
| UV Index | 0–11+ |
| สภาพอากาศโดยรวม | เช่น ท้องฟ้าแจ่มใส, มีเมฆ, ฝนตก |

### F-05 แปลงพิกัดเป็นชื่อสถานที่ (OpenStreetMap Nominatim)
- แสดงชื่อหมู่บ้าน / ตำบล / จังหวัด ที่แปลงตั้งอยู่
- ใช้ในรายงานแทนการแสดงพิกัดตัวเลข

### F-06 วินิจฉัยด้วย AI (Gemini)

AI วิเคราะห์ภาพพืชร่วมกับข้อมูลอากาศและพิกัด แยกผลเป็น **5 หมวด**:

| หมวด | รหัส | ตัวอย่าง |
|------|------|---------|
| โรคพืช | `disease` | โรคใบไหม้, ราน้ำค้าง, แอนแทรคโนส |
| แมลงศัตรูพืช | `pest` | เพลี้ยกระโดด, หนอนกอ, ไรแดง |
| สภาพอากาศ | `weather` | ขาดน้ำ, น้ำท่วม, เครียดความร้อน |
| ธาตุอาหารพืช | `nutrient` | ขาด N, ขาด Fe, ขาด Mg |
| พิษสารเคมี | `chemical` | ยาตกค้าง, สารกำจัดวัชพืชเกินขนาด |

ผลลัพธ์ต่อ 1 การวินิจฉัย:
- ชื่อโรค/ปัญหา (ไทย + อังกฤษ + ชื่อวิทยาศาสตร์)
- ระดับความรุนแรง 1–5
- ความมั่นใจ (%)
- อาการที่สังเกตเห็น
- เหตุผลการวินิจฉัย
- คำแนะนำการแก้ไข
- **แหล่งอ้างอิงทางวิชาการ** (FAO, กรมวิชาการเกษตร, IRRI, CABI)

### F-07 บันทึกลง Google Drive
- สร้างโฟลเดอร์อัตโนมัติ: `ฟาร์ม / ปี / เดือน / ชื่อแปลง`
- อัปโหลดภาพถ่ายทุกใบ
- สร้างไฟล์สรุป `.md` คู่กับภาพ
- บันทึกแถวข้อมูลลง Google Sheets

---

## 4. รูปแบบข้อมูลที่รับเข้า

```
Input ต่อ 1 การตรวจ:
  ├── images[]         รูปภาพพืช (1–5 ไฟล์)
  ├── plot_id          รหัสแปลง
  ├── plot_name        ชื่อแปลง (optional)
  ├── latitude         พิกัด GPS
  ├── longitude        พิกัด GPS
  ├── gps_accuracy_m   ความแม่นยำ GPS (เมตร)
  ├── timestamp        วันเวลา (ISO 8601)
  ├── inspector_name   ชื่อผู้ตรวจ
  └── notes            หมายเหตุเพิ่มเติม (optional)
```

---

## 5. รูปแบบเอกสารสรุป (.md) ที่บันทึกลง Drive

```
# รายงานการตรวจแปลง — [ชื่อแปลง]

## ข้อมูลการตรวจ
- ผู้ตรวจ / วันเวลา / พิกัด / สถานที่

## สภาพอากาศ ณ เวลาตรวจ
- อุณหภูมิ, ความชื้น, ฝน, ลม, UV

## ผลวินิจฉัย AI
- ตาราง 5 หมวด (พบ / ไม่พบ)
- รายละเอียดแต่ละปัญหา + คำแนะนำ + อ้างอิง

## รูปภาพ
- link ไปยังไฟล์ใน Drive
```
